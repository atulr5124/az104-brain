# Azure ExpressRoute

Azure ExpressRoute lets you extend your on-premises networks into the
Microsoft cloud over a private connection facilitated by a connectivity
provider. With ExpressRoute, you can establish connections to Microsoft
cloud services, such as Microsoft Azure and Microsoft 365.

## Key Features

- Private connectivity — traffic does not traverse the public internet
- More reliable than typical internet connections
- Lower latency than internet connections
- Higher security than internet connections
- Speeds from 50 Mbps to 100 Gbps

## ExpressRoute vs VPN Gateway

ExpressRoute provides dedicated private connectivity while VPN Gateway
uses encrypted tunnels over the public internet. ExpressRoute is more
expensive but offers better reliability, speed, and security guarantees.
Use ExpressRoute for mission-critical workloads that need consistent
performance. Use VPN Gateway for lower-cost connectivity where occasional
latency spikes are acceptable.

## Exam Focus Points

- ExpressRoute does NOT traverse the public internet
- Connectivity is through a third-party provider
- Supports BGP routing
- Two circuits can be provisioned for high availability
- ExpressRoute Global Reach connects on-premises networks through Microsoft
